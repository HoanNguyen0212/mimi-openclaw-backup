## Business Logic & System Architecture (2026-03-11 09:05 UTC)
- **Decision on Data Storage:** Sếp confirmed the "Text-First" philosophy (Text > Brain). We will NOT create custom SQL tables in PostgreSQL for work management.
- **Markdown as Database:** All work data (contacts, tasks, case details) will be stored in Markdown files (`.md`) within the `workspace`.
- **Planned Work Management System:**
    - **Storage:** Markdown files in `memory/` or `projects/` will be the primary source of truth.
    - **Processing:** Python scripts (using `pandas`, `python-docx`, `openpyxl`) will parse these Markdown files to generate `.docx` or `.xlsx` reports.
    - **Integration:** This maintains system simplicity and portability while providing structured reporting capabilities.
